Stack Canary - метод защиты кода от переполнения с целью перезаписать адрес возврата из функции и перехода к любому места кода программы.

В конце перед ret происходит проверка локальной переменной, созданной в начале `[rbp+canary]` и канарейкой через xor, если одинаковы, то будет 0 (a xor a == 0), произойдёт переход на loc_4009C0, иначе вызовется функция `__stack_chk_fail`, которая завершит работу программы с ошибкой.

# Способы обхода
## Не перетирать
Если есть уязвимость получения доступа к любому участку памяти через массив.

## Уязвимость форматной строчки
`printf("dump: %p %p %p %p %p")`
`%p` - формат вывода указателя.

`printf("dump: %15$p %16$p %17$p %18$p %19$p")`
`%15$p` - вывод элемента по индексу. В первую очередь возьмёт значения из регистров, а затем пойдёт по стеку.

**Пример:**
`printf("dump: %15$p %16$p %17%p")`
**Вывод:**
dump: 0xb7f99848d5e9aa00 0x7ffdf921bf40 0x401203
первое - канарейка
второе - сохранённый стекфрейм
третье - адрес возврата.

Это возможно, если есть контроль над форматной строчкой:
``` c++
char buf[64];
read(0, buf, 64);
printf(buf);
```

## Брутфорс
Основан на особенности работы сетевых сервисов. Сервер слушает подключения и при новом подключении создаёт форк основной программы (копия процесса), при этом значение канарейки не меняется.

**Использование:**
Перетирание канарейки по чуть-чуть.
Например, канарейка = 0x17df88e6e557f00.
Для нас выглядит как 0xXXXXXXXXXXXXX00, где X - неизвестная цифра. А XX - 1 байт.

Брутфорс осуществляется через подбор канарейки по одному байту:
1 попытка: пробуем 0xXXXXXXXXXXX0100 - ошибка
2 попытка: пробуем 0xXXXXXXXXXXX0200 - ошибка..
128 попытка: пробуем 0xXXXXXXXXXXX7f00 - программа завершила работу без ошибки, успешно подобран 1 байт, так необходимо подобрать всю.

## Напечатать
Функция `printf` печатает всё, пока не встретит нуль байт (`\00`), если имеется возможность переполнить буфер и затереть первый байт канарейки, то она выведется вместе с буфером.
