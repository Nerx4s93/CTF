Shellcode - бинарный кусок кода, который засылается в программу. Набор ассемблерных инструкций в байт-коде.
Набор готовых шелл-кодов: [Shellcodes database for study cases](https://shell-storm.org/shellcode/index.html)

# Обход ограничений копирования
Например, при использовании функции `strcpy` не получится записать шел-код, в котором присутствуют нуль-байты.

Шел-код, который не сможет записаться в память с `strcpy`:
``` nasm
mov rax, 4
;\xb8\x04\x00\x00\x00
```
Для обхода этого можно использовать `xor` и `al`:
``` nasm
xor rax, rax
mov al, 4
; \x48\x31\xc0\xb0\x04
```

Если всё таки необходимо сделать так, чтобы значение в регистре имело нуль байт:
``` nasm
mov si, 0x1000 ; \x66\xbe\x00\x10
```
Можно сделать его не нулевым, а затем отнять:
``` nasm
xor rsi, rsi
mov si, 0x1001
dec rsi
; \x48\x31\xf6\x66\xbe\x01\x10\x48\xff\xce
```

# Обход ограничений по памяти
Необходимо сократить размер шел-кода в байт коде.

## Использование другого набора инструкций
### Закольцовывание регистров
``` nasm
mov r9, 0xffffffffffffffff
; \x49\xc7\xc1\xff\xff\xff\xff
```

``` nasm
xor r9, r9
dec r9
; \x4d\x31\xc9\x49\xff\xc9
```

# Использовать lea
Если предварительно необходимо было занулить регистр, то это экономит 2 байта.
``` nasm
; До этого был xor rax, rax
lea r9, [rax-1]
; \x4c\x8d\x48\xff
```
