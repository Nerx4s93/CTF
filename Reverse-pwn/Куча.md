# C++ классы
**Без виртуальных методов (Plain)**:
Объект — это просто структура с данными.
- **Смещение `+0`:** Первое поле данных.
- **Смещение `+N`:** Остальные поля в порядке объявления.
- _Размер:_ Сумма размеров всех полей + байты выравнивания (padding).
**С виртуальными методами (Polymorphic)**:
В начало добавляется скрытый указатель для работы полиморфизма.
- **Смещение `+0`:** **vptr** (8 байт) — адрес таблицы виртуальных функций (`vtable`).
- **Смещение `+8`:** Первое поле данных.
- **Смещение `+8+N`:** Остальные поля.

# Type Confusion
**Type Confusion** - это уязвимость, возникающая, когда программа обращается к объекту одного типа, используя методы или указатели, предназначенные для другого типа.
Программа не проверяет (или проверяет некорректно) соответствие типа объекта перед его использованием. Это приводит к нарушению типизации и неверной интерпретации содержимого памяти.
Если программа ожидает объект класса `Person`, но вместо него подсовывается объект `Meter`, функции `Person` могут принять данные из `Meter` за адреса в памяти или указатели на таблицы виртуальных функций (vtable).
Пример: [Accounter](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/Accounter.md), [Confusion Hall](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/Confusion%20Hall.md).

# Use-After-Free
**Use-After-Free** — это уязвимость, возникающая при попытке обращения к объекту через указатель после того, как память, на которую он ссылается, была помечена как свободная (освобождена).
**Type Confusion (через UAF):** возникает при аллокации нового объекта со специфической структурой в тот же адрес. При обращении по старому указателю программа интерпретирует данные нового объекта как поля исходного.
Пример: [Liberal Executor](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/Liberal%20Executor.md), [UAF Fun House](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/UAF%20Fun%20House.md).

# Double Free
**Double Free** — уязвимость, возникающая при повторном вызове функции освобождения памяти (`free`) для одного и того же указателя без промежуточной аллокации.
**Type Confusion (через Double Free):** возникает при зацикливании связного списка аллокатора (fastbins/tcache), что позволяет дважды получить один и тот же адрес через `malloc`.
Но на новых версиях libc есть защита от double free.
Метод обхода защиты:
1. Аллоцировать 9 разных объектов.
2. Освободить память 7-ми объектов.
3. Освободить: №8, №9, №8.
4. Аллоцировать 7 ненужных объектов.
5. Аллоцировать A, B, C.
По итогу A и С указывают на одну и ту же память.
Пример: [Wishlist](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/Wishlist.md), [Just2Free](../2.%20Таски/pwn/pwn.spbctf.ru/8.%20Type%20Confusion,%20Use-After-Free,%20Double%20Free/Just2Free.md).
