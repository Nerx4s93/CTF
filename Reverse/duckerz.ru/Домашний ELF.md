# Анализ кода

## Точка входа
![photo_2026-03-03_19-25-20](../../z.%20Images/photo_2026-03-03_19-25-20.jpg)
Красным цветом выделен блок кода, которые отвечает за вызов системной функции вывода в консоль.
Фиолетовым цветом выделен блок кода, в котором вызывается системная функция чтения данных из консоли. Результат записывается в регистр ecx.
В оранжевом блоке происходит побайтовый xor введённой строки на 0x34=52. 
	`ecx[ebx] ^= 52`

После xor идёт вызов функции sub_80480DA, после результат функции сравнивается с 0. Если eax (результат функции) == 0, то программа переходит в loc_80480B6, где в регистр ecx помещается сообщение об правильном ключе и программа завершает работу.

## Функция sub_80480DA
Функция имеет две точки выхода, первая на метке loc_80480FC, а вторая на метке loc_8048103. Т.к. чтобы вывелось нужное нам сообщение, необходимо, чтобы при выходе из функции значение регистра eax было равно 0 необходима, чтобы функция совершила переход на метку loc_80480FC.
![{F8A869D1-1B0A-45D2-B429-2F8B44C5EAD8}](../../z.%20Images/{F8A869D1-1B0A-45D2-B429-2F8B44C5EAD8}.png)

Аргументы функции
![{929489F6-8A99-4289-B3DB-DA381718D82C}](../../z.%20Images/{929489F6-8A99-4289-B3DB-DA381718D82C}.png)
Первый аргумент - введённая строка xor 52 (ecx).
Второй аргумент - edx, указывает на зашифрованное сообщение.
Третий аргумент 0Ch - количество символов (12 байт), не 13, т.к. последний символ - \n.
